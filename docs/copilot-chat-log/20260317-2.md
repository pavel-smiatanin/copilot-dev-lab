### Result
Implemented task 6 (SSRF protection for destination URL validation) and updated the backlog.

**Files created:**
- `backend/src/UrlShortener.Shared/Url/PrivateNetworkGuard.cs` — static utility class providing SSRF protection:
  - `IsPrivateOrReserved(IPAddress ip)` — pure IP range check (no I/O). Covers: IPv4/IPv6 loopback, RFC 1918 (10/8, 172.16/12, 192.168/16), link-local (169.254/16, fe80::/10), CGNAT (100.64/10), reserved (0/8, 240/4), IPv6 unique local (fc00::/7), IPv4-mapped IPv6 normalization.
  - `IsHostAllowedAsync(string host, CancellationToken)` — resolves host via DNS and returns `false` if ANY resolved address is private/reserved; fails closed on DNS failure.
- `backend/tests/UrlShortener.Shared.Tests/Url/PrivateNetworkGuardTests.cs` — 51 unit tests covering all IP range categories, boundary values, IPv4-mapped IPv6, localhost hostname, and unresolvable hostname.

**Files modified:**

The rule runs only when the URL scheme is already valid (via `.When(x => IsValidHttpUrl(x.DestinationUrl))`), preventing unnecessary DNS lookups on malformed URLs. Returns 400 on violation via the existing `ValidationBehavior` pipeline.
- `docs/backlog.md` — marked tasks 4, 5, and 6 as `[x]`.

**Test results:** 51/51 passed in `UrlShortener.Shared.Tests`.

**Key design decisions:**
- SSRF check placed in FluentValidation (`MustAsync`) for consistent 400 responses via the existing pipeline behavior.
- Fails closed: DNS failure blocks the request (security over availability).
- Blocks if ANY resolved IP is private (prevents split-DNS attacks).
- `PrivateNetworkGuard` is a pure static utility in `UrlShortener.Shared` for reuse by task 7 (metadata prefetch).
