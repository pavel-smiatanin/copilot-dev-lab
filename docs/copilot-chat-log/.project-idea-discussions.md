### Prompt
I like the idea for the 4th project Simple URL Shortener. Compose detailed list of user flows, functional features and requirements the project should have. Save the results in /docs/implementation/srs.md

### Response

**User Flows**
- **Anonymous Create Short Link:** User opens SPA → enters destination URL → optional custom alias/expiry/password → passes captcha if rate‑limited → submits → system validates URL/alias, creates shortId, returns shortUrl and copy action.  
- **Authenticated Create & Save:** User signs in (OIDC/Google or email) → opens dashboard → clicks New Link → enters details (title, tags, custom alias, expiry, visibility) → preview metadata fetched → save → link appears in user list with actions.  
- **Custom Alias Collision Handling:** User requests alias → system checks uniqueness → if taken show conflict and suggestions (random, suffix, reuse existing per user) → user retries or accepts suggestion.  
- **Password‑Protected Link Access:** Visitor requests /:alias → system checks enabled/expiry → if password required show unlock form → user posts password → system verifies hash → on success redirect and record visit token; failures show 403 after N attempts.  
- **Redirect Flow (fast path):** Public visitor hits /:alias → Redis/cache lookup of alias→ if found and enabled return 302 to destination; asynchronously enqueue visit record; if missing fallback to DB lookup then cache.  
- **Manage Links (authenticated):** User opens dashboard → list with filters/sort/pagination → can edit metadata, add/remove tags, disable/enable, view analytics, export single link data, soft‑delete (trash) and restore.  
- **Bulk Import / Export:** Authenticated user uploads bookmarks HTML/CSV → mapping UI shows preview and errors → user confirms → server processes in background; export selected or all links to CSV/JSON with tags and metadata.  
- **Analytics View:** User opens link details → aggregated stats (total clicks, uniques, recent timeline, top referrers/countries/devices) are fetched; raw visits accessible with pagination and IP hashing.  
- **API Key / Programmatic Creation:** Developer generates API key → configures scopes/rate limits → calls POST /api/links with API key header → receives shortUrl; keys can be revoked.  
- **Admin Moderation:** Admin logs in → views flagged/suspicious links → can disable link, add domain to blacklist, revoke API keys, view global metrics and export logs.

**Functional Features**
- **Authentication & Accounts:** OIDC (Google), optional local email, JWT refresh, account linking, profile.  
- **Link CRUD:** Create (anon/auth), read, update (title, destination, expiry, tags, password, visibility), soft delete + restore, hard delete.  
- **Alias Management:** Auto-generated shortId, custom alias with validation and collision handling, allowed char set and length rules.  
- **Redirect Handling:** Fast cache + DB fallback; 302 redirect default; password/unlock flow support; expiry/enabled checks.  
- **Analytics & Visits:** Record visit (timestamp, hashed IP, referrer, UA, country), aggregated stats endpoint, CSV export, visit retention policy.  
- **Search & Filters:** Search by title/URL/alias; filter by date, tags, status, owner, domain.  
- **Tags & Metadata:** Optional tags, auto‑fetch page metadata (title, favicon, og:image), user tags and tag suggestions.  
- **Bulk Operations:** Import (HTML/CSV) with mapping, bulk tagging, bulk delete/export.  
- **API & Integrations:** REST CRUD, API keys, rate limiting, CORS, webhook support for link create/visit.  
- **Admin Controls:** Blacklist domains/patterns, moderate links, audit logs, user management.  
- **Extensions & Capture:** Bookmarklet/browser extension to capture URL + selection; clipboard copy actions; QR code generation.  
- **Security & Privacy Features:** Password protection, per‑link visibility, IP hashing, account deletion and data export.

**Functional Requirements (concrete)**
- **Unique IDs & Aliases:** Generated shortId length configurable (6–10); custom alias uniqueness enforced at DB level.  
- **Validation:** Destination must be valid http/https URL; block disallowed schemes and private IP ranges during metadata fetch.  
- **Redirect Speed:** Alias resolution should use Redis cache; redirects must not block on analytics writes.  
- **Rate Limits & Abuse:** Anonymous: low rate limit + captcha; authenticated: higher quota; API keys: scoped limits.  
- **Persistence:** Use Postgres for core entities; visits in partitioned table or append store; Redis for alias cache and rate counters.  
- **Password Storage:** Use salted bcrypt/Argon2 hashes for link passwords.  
- **Visibility Rules:** Private links accessible only by owner or via token; public links resolvable by anyone.  
- **Duplicate Detection:** Suggest existing user link for same normalized destination URL.  
- **Soft Delete & Retention:** Soft delete moves to trash for configurable retention (e.g., 30 days) before permanent deletion.  
- **Custom Domains:** Support optional mapping for user custom domains with validation and ownership verification.

**Non‑functional & Operational Requirements**
- **Performance:** Redirect latency target <100ms p95; cache hit rate target >95%.  
- **Scalability:** Stateless app servers, DB read replicas, visits partitioning or event pipeline (Kafka) for high volume.  
- **Availability:** Health checks, graceful degradation of analytics if backend slow.  
- **Backups & Recovery:** Regular DB backups and tested recovery plan.  
- **Observability:** Request tracing, metrics (redirects/sec, errors, latency), alerts for spikes or failures.  
- **CI/CD:** Automated tests, linting, deploy pipelines, canary/blue‑green for production.

**Security & Privacy Requirements**
- **Input Sanitization:** Sanitize metadata and titles to avoid XSS.  
- **SSRF/Metadata Fetch Safety:** Resolve destination host and block private IP ranges before fetching.  
- **HTTPS & HSTS:** Enforce TLS for all endpoints and cookies.  
- **CSRF Protection:** If cookies used for auth, protect write endpoints.  
- **IP Handling:** Store hashed/truncated IPs; provide opt‑in for full IP if justified and documented.  
- **Audit & Access Logs:** Record admin actions, API key usage, suspicious activity flags.  
- **Data Portability & Erasure:** Export all user data (JSON/CSV) and complete deletion flow.

**Data Model & Storage Suggestions**
- **Core tables/collections:** users, links, visits, api_keys, blacklist, tags, audit_logs.  
- **Indexing:** Unique index on short_id/custom_alias; index visits on link_id + timestamp; owner_id + created_at for listing.  
- **Cache keys:** `alias:{shortId}` -> destination metadata; `user:quota:{userId}` for rate counters.

**Testing & Acceptance Criteria**
- **Unit tests:** ID generation, alias validation, URL normalization, auth logic.  
- **Integration tests:** Create link → GET shortUrl redirects; password flow; expiry handling; custom alias conflicts.  
- **E2E tests:** SPA flows: sign-in, create link, follow link, analytics view, bulk import.  
- **Load tests:** Redirect throughput and latency under expected peak; ensure analytics pipeline decoupling prevents tail latency.  
- **Security tests:** SSRF checks, open redirect validations, XSS sanitization tests, API key misuse scenarios.
